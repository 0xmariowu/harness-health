# Security Policy

## Supported versions

| Version | Supported |
|---------|-----------|
| X.Y.x   | Yes       |
| X.Y.x   | Yes       |
| < X.Y   | No        |

## Reporting a vulnerability

Please report security issues via GitHub's private advisory form instead of a
public issue:

  https://github.com/__OWNER__/__PROJECT_NAME__/security/advisories/new

If advisories are not enabled or you don't have a GitHub account, email the
maintainer listed in `.github/CODEOWNERS` directly.

We will acknowledge receipt within 48 hours.

## Scope

Security concerns include:

- Authentication or authorization bypass
- Data exposure or leakage
- Command injection or code execution
- API key or secret exposure in logs or artifacts
